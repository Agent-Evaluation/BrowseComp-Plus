The BrowseComp-Plus evaluation framework is orchestrated through a TypeScript runtime that interfaces directly with a robust Python-based BM25 retrieval subprocess to measure single-agent and multi-agent reasoning performance over large web corpora. At the core of the system lies a BM25 bridge that spawns a long-lived Python environment utilizing Pyserini and PyJNIus to leverage Java's Lucene search capabilities. The main runtime communicates with this subprocess asynchronously through standard input and output streams using structured JSONL messages, creating an architectural pattern that circumvents the overhead of repeatedly bootstrapping the Python interpreter and index loaders across evaluation queries, ensuring highly scalable retrieval. For a baseline single-agent configuration, the runtime provisions an iterative search-and-reason loop where a solitary specialized agent accesses this bridge to repeatedly gather and synthesize evidence until a response is finalized. Conversely, the multi-agent system evaluation encapsulates distinct hierarchical and collaborative topologies—such as independent, centralized, decentralized, and hybrid configurations. In these multi-agent workflows, independent specialized agents execute parallel retrieval through the shared BM25 bridge to formulate initial proposals based on unique search strategies. Depending on the active topology, these proposals are subsequently iteratively debated among peer agents or synthesized by a centralized leader agent. Once an agent or group of agents reaches consensus, or simply exhausts its prescribed tool-call and iteration budget, the final textual answer alongside trace metrics are serialized. Following execution, an offline evaluation script queries a large language model acting as an automated judge, such as MoonshotAI's Kimi-K2.5, to assess prediction accuracy, compute calibration errors, and tabulate average tool usage against a predefined ground truth dataset.
